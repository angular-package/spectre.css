Spectre.css is designed with consistent design language. You can use your variables to create your new or match an existing design. All variables in _variables.scss include !default flag. You can redefine the variables to override the values.

Almost all the Sass variables are used to initialize equivalent CSS variables, means, they are defined in the same pattern as Sass variables to have naming consistency, but with an additional customizable prefix.

Variables - Prefix#

The Spectre.css has the Sass $var-prefix variable that is used as the prefix of the CSS variables, and as the prefix of some CSS classes(this feature replaces in some minimal way custom prefixes for all classes with the gulp). In general CSS variable name structure is defined as follows. 

--#{$var-prefix}-#{$name}: #{$value}

Click on the link for detailed information.

Technical documentation

Variables - Sizes#

The default REM (root em) size in Spectre.css is 20px.

Please note in the Chinese/Japanese/Korean version, Chrome uses 12px as default minimum font size. That is why Spectre.css sets rem default html font size to 20px, which is relatively easy to calculate.

Spectre.css uses 4px as the unit size. Sizes of components are based on the unit size.

Variables - Color scheme#

The CSS color-scheme property is set to normal in both :root and :host elements with the Sass $color-scheme variable.

// src/_variables.scss
$color-scheme: normal;

Variables - Colors#

Spectre.css has some colors with defined hex and colors that depend on them, which means, for example, if the primary-color is overridden, then dependent primary-dark and primary-light automatically change.

Each hex color is in hsla() form defined by the define-color() mixin, split into four CSS variables, where suffix h indicates hue, s saturation, l lightness and a alpha. Click here for detailed information.

The single CSS color is defined as follows.

--#{$var-prefix}-#{$name}-color-h: hue($color)
--#{$var-prefix}-#{$name}-color-s: saturation($color)
--#{$var-prefix}-#{$name}-color-l: lightness($color)
--#{$var-prefix}-#{$name}-color-a: alpha($color)

Variables - Hero sizes#

These are common variables for the hero element defined in the _variables.scss via the sass map $hero-size variable.

xs sm md lg xl 2x 
// src/_variables.scss
// Hero sizes.
$hero-size: (
  'xs': 1rem, // --#{$var-prefix}-hero-size-xs.
  'sm': 2rem, // --#{$var-prefix}-hero-size-sm.
  'md': 4rem, // --#{$var-prefix}-hero-size-md.
  'lg': 8rem, // --#{$var-prefix}-hero-size-lg.
  'xl': 10rem // --#{$var-prefix}-hero-size-xl.
) !default;

Specific color hero sizes

There are hero sizes CSS variables defined only for the specific colors giving the ability to set different sizes for one color. For example, there is a .hero-primary class that fills the hero with the --s-primary-color, and to change the size assigned to the primary color to large add class hero-primary-size-xl. Read more about hero sizes customization here.
